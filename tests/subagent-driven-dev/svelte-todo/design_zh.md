# Svelte 待办事项清单 - 设计

## 概述

一个使用 Svelte 构建的简单待办事项清单应用。支持创建、完成和删除待办事项，并具备 localStorage 持久化功能。

## 功能特性

- 添加新待办事项
- 标记待办事项为完成/未完成状态
- 删除待办事项
- 筛选选项：全部 / 未完成 / 已完成
- 清除所有已完成事项
- 数据持久化至 localStorage
- 显示剩余事项数量

## 用户界面

```
┌─────────────────────────────────────────┐
│  Svelte 待办事项                        │
├─────────────────────────────────────────┤
│  [________________________] [添加]      │
├─────────────────────────────────────────┤
│  [ ] 购买食材                      [x] │
│  [✓] 遛狗                         [x] │
│  [ ] 编写代码                      [x] │
├─────────────────────────────────────────┤
│  剩余 2 项                              │
│  [全部] [未完成] [已完成]  [清除已完成] │
└─────────────────────────────────────────┘
```

## 组件结构

```
src/
  App.svelte           # 主应用，状态管理
  lib/
    TodoInput.svelte   # 文本输入框 + 添加按钮
    TodoList.svelte    # 列表容器
    TodoItem.svelte    # 单个待办项（含复选框、文本、删除按钮）
    FilterBar.svelte   # 筛选按钮 + 清除已完成功能
    store.ts           # Svelte 状态存储（待办事项）
    storage.ts         # localStorage 持久化模块
```

## 数据模型

```typescript
interface Todo {
  id: string;        // 唯一标识符
  text: string;      // 待办事项文本
  completed: boolean;
}

type Filter = 'all' | 'active' | 'completed';
```

## 验收标准

1. 可通过输入后按回车键或点击“添加”按钮来新增待办事项
2. 可通过点击复选框切换待办事项的完成状态
3. 可通过点击 X 按钮删除单个待办事项
4. 筛选按钮能正确显示对应分类的待办事项
5. “剩余 X 项”准确显示未完成事项的数量
6. “清除已完成”功能可移除所有已完成事项
7. 待办事项在页面刷新后保持持久化（localStorage）
8. 空状态时显示友好的提示信息
9. 所有测试用例通过
