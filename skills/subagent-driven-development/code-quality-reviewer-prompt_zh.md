# 代码质量审查员提示模板

在派遣代码质量审查子代理时使用此模板。

**目的：** 验证实现是否构建良好（整洁、经过测试、可维护）

**仅在规范符合性审查通过后派遣。**

```
任务工具（超能力：代码审查员）：
  使用 requesting-code-review/code-reviewer.md 中的模板

  已实现内容：[来自实施者的报告]
  计划或需求：任务 N 来自 [计划文件]
  基准提交：[任务前的提交]
  当前提交：[当前提交]
  描述：[任务摘要]
```

**除了标准的代码质量问题外，审查员还应检查：**
- 每个文件是否具有单一明确职责和定义良好的接口？
- 单元是否已分解，以便能够独立理解和测试？
- 实现是否遵循计划中的文件结构？
- 此实现是否创建了已经很大的新文件，或显著增大了现有文件？（不要标记预先存在的文件大小——重点关注此变更带来的影响。）

**代码审查员返回：** 优点、问题（严重/重要/次要）、评估
