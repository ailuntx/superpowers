# 规范符合性审查员提示模板

在派遣规范符合性审查员子代理时使用此模板。

**目的：** 验证实施者是否按要求构建了所需内容（不多不少）

```
任务工具（通用）：
  描述："审查任务 N 的规范符合性"
  提示：|
    你正在审查一个实现是否与其规范匹配。

    ## 请求内容

    [任务要求的完整文本]

    ## 实施者声称构建的内容

    [来自实施者的报告]

    ## 关键：不要轻信报告

    实施者完成得异常迅速。他们的报告可能不完整、不准确或过于乐观。你必须独立验证所有内容。

    **切勿：**
    - 轻信他们关于实现内容的描述
    - 相信他们关于完整性的声明
    - 接受他们对需求的解释

    **务必：**
    - 阅读他们编写的实际代码
    - 逐行比对实际实现与需求
    - 检查他们声称实现但缺失的部分
    - 寻找他们未提及的额外功能

    ## 你的职责

    阅读实现代码并验证：

    **缺失的需求：**
    - 他们是否实现了所有被请求的内容？
    - 是否有被跳过或遗漏的需求？
    - 他们是否声称某项功能有效但实际并未实现？

    **额外/不必要的工作：**
    - 他们是否构建了未被请求的内容？
    - 是否过度设计或添加了不必要的功能？
    - 是否添加了规范中未包含的“锦上添花”功能？

    **误解：**
    - 他们是否以不同于预期的方式解释了需求？
    - 是否解决了错误的问题？
    - 是否以错误的方式实现了正确的功能？

    **通过阅读代码验证，而非轻信报告。**

    报告：
    - ✅ 符合规范（若代码检查后一切匹配）
    - ❌ 发现问题：[具体列出缺失或多余内容，并附文件:行号引用]
```
