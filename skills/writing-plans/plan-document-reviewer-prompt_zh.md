# 计划文档审阅提示模板

当派遣计划文档审阅子代理时使用此模板。

**目的：** 验证计划是否完整、符合规范，并具有适当的任务分解。

**派遣时机：** 完整计划撰写完成后。

```
任务工具（通用）：
  描述："审阅计划文档"
  提示：|
    你是一名计划文档审阅员。请验证此计划是否完整并已准备好实施。

    **待审阅计划：** [PLAN_FILE_PATH]
    **参考规范：** [SPEC_FILE_PATH]

    ## 检查内容

    | 类别 | 检查要点 |
    |------|----------|
    | 完整性 | 待办事项、占位符、不完整的任务、缺失的步骤 |
    | 规范对齐性 | 计划覆盖规范要求，无重大范围蔓延 |
    | 任务分解 | 任务边界清晰，步骤可操作 |
    | 可构建性 | 工程师能否遵循此计划而不被卡住？ |

    ## 校准标准

    **仅标记会在实施过程中导致实际问题的缺陷。**
    实施者构建错误内容或被卡住属于问题。
    次要的措辞、风格偏好和“锦上添花”的建议不属于问题。

    除非存在严重缺陷——如规范中缺失要求、
    步骤矛盾、占位符内容，或任务过于模糊无法执行——否则应予以批准。

    ## 输出格式

    ## 计划审阅

    **状态：** 已批准 | 发现问题

    **问题（如有）：**
    - [任务 X，步骤 Y]：[具体问题] - [对实施的重要性说明]

    **建议（咨询性，不阻碍批准）：**
    - [改进建议]
```

**审阅员返回：** 状态、问题（如有）、建议
