# 文档评审系统实施计划

> **面向智能体工作者：** 必需：使用超能力：子智能体驱动开发（如果子智能体可用）或超能力：执行计划来实施此计划。

**目标：** 在头脑风暴和撰写计划技能中添加规范和计划文档评审循环。

**架构：** 在每个技能目录中创建评审员提示模板。修改技能文件，在文档创建后添加评审循环。使用Task工具配合通用子智能体进行评审员调度。

**技术栈：** Markdown技能文件，通过Task工具调度子智能体

**规范：** docs/superpowers/specs/2026-01-22-document-review-system-design.md

- [ ] **步骤 1：** 创建评审员提示模板文件

```markdown
# 规范文档评审员提示模板

在调度规范文档评审员子智能体时使用此模板。

**目的：** 验证规范是否完整、一致，并已准备好进行实施规划。

**调度时机：** 规范文档写入 docs/superpowers/specs/ 之后

```
Task tool (general-purpose):
  description: "评审规范文档"
  prompt: |
    你是一名规范文档评审员。验证此规范是否完整并已准备好进行规划。

    **待评审的规范：** [SPEC_FILE_PATH]

    ## 检查内容

    | 类别 | 检查要点 |
    |----------|------------------|
    | 完整性 | TODOs、占位符、"TBD"、不完整的章节 |
    | 覆盖度 | 缺失的错误处理、边界情况、集成点 |
    | 一致性 | 内部矛盾、冲突的需求 |
    | 清晰度 | 模糊的需求 |
    | YAGNI | 未请求的功能、过度设计 |

    ## 关键检查项

    请特别仔细检查：
    - 任何 TODO 标记或占位符文本
    - 标明"稍后定义"或"当X完成时再规范"的章节
    - 明显比其他章节更简略的章节

    ## 输出格式

    ## 规范评审

    **状态：** ✅ 已批准 | ❌ 发现问题

    **问题（如有）：**
    - [章节 X]: [具体问题] - [重要性说明]

    **建议（咨询性）：**
    - [不阻碍批准的建议]
```

**评审员返回：** 状态、问题（如有）、建议
```

```markdown
**规范评审循环：**
编写规范文档后：
1. 调度 spec-document-reviewer 子智能体（参见 spec-document-reviewer-prompt.md）
2. 如果 ❌ 发现问题：

**评审循环指导：**
- 由编写规范的同一智能体修复问题（保持上下文）
- 如果循环超过 5 次，则向人类寻求指导
- 评审员是咨询性的 - 如果你认为反馈不正确，请解释分歧点
```

## 模块 2：计划文档评审员

此模块将计划文档评审员添加到撰写计划技能中。

### 任务 3：创建计划文档评审员提示模板

**文件：**
- 创建：`skills/writing-plans/plan-document-reviewer-prompt.md`

- [ ] **步骤 1：** 创建评审员提示模板文件

```markdown
# 计划文档评审员提示模板

在调度计划文档评审员子智能体时使用此模板。

**目的：** 验证计划模块是否完整、符合规范，并具有适当的任务分解。

**调度时机：** 每个计划模块编写完成后

```
Task tool (general-purpose):
  description: "评审计划模块 N"
  prompt: |
    你是一名计划文档评审员。验证此计划模块是否完整并已准备好实施。

    **待评审的计划模块：** [PLAN_FILE_PATH] - 仅限模块 N
    **供参考的规范：** [SPEC_FILE_PATH]

    ## 检查内容

    | 类别 | 检查要点 |
    |----------|------------------|
    | 完整性 | TODOs、占位符、不完整的任务、缺失的步骤 |
    | 规范对齐 | 模块覆盖了相关的规范要求，无范围蔓延 |
    | 任务分解 | 任务原子化、边界清晰、步骤可操作 |
    | 任务语法 | 任务和步骤使用复选框语法 (`- [ ]`) |
    | 模块大小 | 每个模块不超过 1000 行 |

    ## 关键检查项

    请特别仔细检查：
    - 任何 TODO 标记或占位符文本
    - 仅说明"类似于 X"而无实际内容的步骤
    - 不完整的任务定义
    - 缺失的验证步骤或预期输出

    ## 输出格式

    ## 计划评审 - 模块 N

    **状态：** ✅ 已批准 | ❌ 发现问题

    **问题（如有）：**
    - [任务 X，步骤 Y]: [具体问题] - [重要性说明]

    **建议（咨询性）：**
    - [不阻碍批准的建议]
```

**评审员返回：** 状态、问题（如有）、建议
```
